# STEP31 攻撃モード・元素変化の正規化

## 目的

元素スキルや元素爆発によって通常攻撃の見た目・元素・倍率参照先・ダメージ扱いが変わるケースを、単純な `elementOverride` から分離する。

## 設計

計算項目は次の三軸を別々に保持する。

- `attackType`: プレイヤーが行う操作。通常攻撃、重撃、落下攻撃など。
- `damageType`: バフ判定に使うダメージ分類。通常攻撃、元素スキル、元素爆発など。
- `element`: 最終的に計算する元素。

専用倍率がスキル／爆発セクションに存在する場合は、その項目に `attackMode` を付与する。通常攻撃セクションの物理倍率へ同じ元素変化を重ねない。専用倍率がない単純元素付与だけ、通常攻撃セクションへ適用する。

`attack-mode-rules.json` は原文で「元素爆発ダメージと見なす」などが明記され、操作種別とダメージ分類が異なるケースだけを登録する。未登録時は従来の分類を維持する。

## 適用優先順位

1. 倍率データに記録された所属セクションと元素
2. `attack-mode-rules.json` の明示的なダメージ分類
3. 条件付き `elementOverride`
4. 入力欄の元素ダメージ補正

元素変化レコードと専用倍率の元素が矛盾する場合は推測適用せず、候補へ隔離する。

## 完了条件

- 雷電将軍の爆発中通常・重撃・落下攻撃が爆発ダメージ補正を参照する。
- ニィロウの剣舞がスキルダメージ補正を参照する。
- タルタリヤなど専用倍率を持つキャラで、基礎通常攻撃へ元素変化が重複しない。
- 綾華など専用倍率を持たない元素付与は従来どおり通常攻撃倍率へ適用される。
- 元素が矛盾する生成レコードは計算へ混入しない。

## 天賦状態カード

天賦補正は補正レコード単位ではなく、通常攻撃・元素スキル・元素爆発・固有天賦の出典単位でまとめる。正式な天賦名、影響対象、共有する状態入力、計算への影響、原文を一つのセクションに表示する。

同じ天賦状態に属する補正は `conditionGroupId` を共有する。たとえば胡桃の「蝶導来世」は、HP参照攻撃力アップと炎元素化を一つのチェックで切り替える。

元素スキル側に専用通常攻撃倍率がある場合は攻撃モードを生成する。状態OFFでは通常攻撃天賦の倍率、ONでは元素スキル側の専用倍率だけを表示し、両方を同時に表示しない。スカークは「七相一閃」を明示的な攻撃モード名として使用する。
