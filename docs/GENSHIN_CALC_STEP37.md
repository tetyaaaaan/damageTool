# STEP37 残存JSON警告の構造化設計

## 目的

STEP34～36完了時点の値情報警告4件を、原文に対応する計算入力へ移行する。警告を抑制するだけの例外は追加しない。

## 武器12515「不滅の騎士道」

- 「四風の詩」は0～4層の共通入力とする。
- 1層ごとの与ダメージ補正は精錬ランク別に7.5/9.5/11.5/13.5/15.5%。
- 「魔導秘儀」は独立した有効・無効入力とする。
- 魔導秘儀が有効な場合だけ、同じ「四風の詩」層数を使って会心ダメージを加算する。
- 同一層数を2回入力させないため、`stack.id` を共有状態キーとして扱う。

## 聖遺物15041・15042

- 月兆状態を `未発動 / 初照 / 満照` の選択入力にする。
- 15041は装備者の会心率へ0/15/30%を適用する。
- 15042は装備者を含むチーム全員の元素熟知へ0/60/120を適用する。
- チーム全体への伝播はSTEP40以降の編成層で再利用し、現段階では装備者自身の計算に適用する。
- 月輝明光1種類ごとの月反応ダメージ+10%は既存の0～4層入力を維持する。

## 完了条件

- `validateCalcData` の警告が0件。
- 武器12515のR1・4層で与ダメージ+30%、魔導秘儀有効時に会心ダメージ+30%。
- 15041満照で会心率+30%、15042満照で元素熟知+120。
- 条件カードで共通層数を重複表示しない。
- 全回帰テストと実画面確認が成功する。
