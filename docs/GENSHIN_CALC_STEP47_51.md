# 原神ダメージ計算 STEP47〜51 設計

## 目的

計算式と攻撃種別の内部構造を維持したまま、計算入力と結果の幅を均等化し、ダメージ値を最優先で比較できる結果表示へ変更する。

## STEP47: レイアウトと表示グループ

- デスクトップでは入力パネルと結果パネルを `1fr 1fr` の等幅にする。
- 980px以下は従来どおり上下1列にする。
- 表示上は通常攻撃・重撃・落下攻撃を `basic` グループへまとめる。
- タブ名は所在が明確な「通常・重撃・落下」とする。
- 内部の `normalAttack` / `chargedAttack` / `plungingAttack` は変更しない。
- 基本攻撃タブ内は「通常攻撃」「重撃」「落下攻撃」のサブタブで分ける。
- 3つのサブタブは該当データがなくても常に表示し、件数はタブ名へ付けない。
- 該当データがないサブタブには、計算可能なデータがないことを明記する。

## STEP48: 結果表

- 各結果を「攻撃名 / 非会心 / 会心 / 平均」の4列で比較できる表にする。
- 元素、攻撃種別、ヒット数は攻撃名の補足情報として小さく表示する。
- 複数ヒットの場合だけ「全ヒット平均」を補助表示する。
- 数値は右揃えとし、桁を追いやすくする。
- シールドなどダメージ以外の結果は専用の単一値表示を維持する。
- 攻撃名の横には小さな「詳細 ▼」だけを置き、内訳行は初期状態で閉じる。
- 同じ表では一度に1件だけ詳細を開き、開いたボタンは「詳細 ▲」へ切り替える。

## STEP49: 利用者向け計算内訳

表示用モデルを計算結果から生成し、以下の順で説明する。

1. 基礎ダメージ
   - 参照ステータス
   - 参照値
   - 天賦倍率
   - 基礎ダメージ
2. ダメージバフ
   - ダメージバフ合計
   - 加算基礎ダメージ
   - 最終ダメージ倍率
   - 適用された効果
3. 元素反応
   - 反応名
   - 反応倍率
   - 元素熟知補正
   - 反応ダメージ補正
4. 敵への補正
   - 防御デバフ
   - 防御無視
   - 最終防御倍率
   - 最終耐性
   - 最終耐性倍率
5. 会心
   - 会心率
   - 会心ダメージ
6. 特殊補正
   - 通常のバフ欄では説明できない固定値、効果置換、特殊乗算

値がゼロで説明不要な項目は省略する。ただし防御倍率と耐性倍率は最終ダメージへの影響を示すため常に表示する。

## STEP50: 表示語・アクセシビリティ・レスポンシブ

- `physical`、`normalAttack`、`chargedAttack` などの内部値を画面へ直接表示しない。
- 元素、攻撃種別、参照ステータス、効果出典は表示用辞書で日本語化する。
- 適用されなかった候補と内部IDは「詳細データ・検証情報」へ分離する。
- タブへ `role=tab`、`aria-selected`、`aria-controls` を付け、左右キーで移動可能にする。
- 狭い画面では攻撃名を上段、非会心・会心・平均を下段の3列にして横スクロールを避ける。
- 計算内訳は各攻撃行の直下へ置き、どの結果の説明かを明確にする。
- 展開後は計算の流れを直接表示し、内部情報は「検証用データを見る」の中へ格納する。

## STEP51: 検証

- 表示グループ統合後も各結果の `entry.attackType` が変わらないこと。
- レンダリング前後で計算結果の数値が変わらないこと。
- `physical` などの内部列挙値が利用者向け内訳へ露出しないこと。
- 複数ヒット時だけ全ヒット平均が表示されること。
- 全Genshinテストを実行すること。
- 実画面で等幅、タブ統合、内訳、日本語表示、モバイル相当幅を確認すること。
