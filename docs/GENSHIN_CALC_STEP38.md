# STEP38 専用元素反応式の設計

## 目的

通常の増幅・加算・変化反応へ代入できない月反応と星電導を、発生経路ごとの専用契約で計算する。

## 計算経路

### 1. 間接月反応

元素付着で発生する月感電と月結晶だけが対象。参加者ごとに次を計算する。

`反応係数 × Lv固有値 × (1 + 月反応基礎ダメージ向上) × (1 + 6×熟知/(熟知+2000) + 反応ダメージ補正) × 耐性補正 × 会心補正`

- 月感電係数: 1.8
- 月結晶係数: 0.96
- 防御補正と通常の元素ダメージバフは使用しない。
- 最大4人の個別値を降順に並べ、`1位 + 2位×1/2 + 3位×1/12 + 4位×1/12` で合算する。
- 期待値は各参加者の会心・非会心全組み合わせを列挙して求める。

### 2. 直接月反応・直接星電導

「月感電/月開花/月結晶/星電導ダメージと見なされる」と明記された天賦項目だけが対象。

`(専用係数 × 天賦倍率 × 参照ステータス × (1 + 基礎ダメージ向上) × (1 + 熟知補正 + 反応ダメージ補正) + 反応加算基礎ダメージ) × 耐性補正 × 会心補正 × 向上補正`

- 月感電: 3.0
- 月開花: 1.0
- 月結晶: 1.6
- 星電導: `1.4 + 0.05 × 極星フィールド蓄積数`（0～12）
- 防御補正と通常の元素ダメージバフは使用しない。
- 天賦項目のラベルと、キャラクター・グループ・entry ID の明示ルールで専用種別を判定する。

### 3. ダメージを発生させない反応

- 月開花そのものは独立した月反応ダメージを発生させない。生成された草原核・豊穣の核は通常の開花系として扱う。
- 星電導そのものは独立した反応ダメージを発生させず、極星フィールドと対応天賦を有効化する。

## 入力UI

- 月感電・月結晶では現在キャラクターを参加者1として入力欄のLv・熟知・会心を自動使用する。
- 参加者2～4は任意入力とし、Lv、元素熟知、会心率、会心ダメージ、反応ダメージ補正、基礎ダメージ向上を入力できる。
- 星電導では極星フィールド蓄積数を0～12で入力する。
- 計算経路、独立ダメージの有無、防御無視、通常ダメージバフ非適用を反応カードへ表示する。

## 補正契約

- `reactionBaseDamageBonus` を解析・計算対象カテゴリへ追加する。
- イネファ、ラウマ、フリンズ、ネフェル、コロンビーナ、茲白、リンネア、サンドローネの固有変換天賦を構造化し、参照ステータスと上限から算出する。
- `reactionBonus` と `reactionCritBonus` は専用反応IDに対してのみ適用する。
- チーム外キャラクターが提供する補正の自動収集は4人編成層（STEP40以降）で行い、STEP38の追加参加者では手入力を正とする。

## 完了条件

- 月感電・月結晶の単独参加・4人寄与・順位入れ替わり・会心期待値をテストする。
- 4種の直接専用反応で係数、熟知、基礎向上、耐性、防御無視をテストする。
- 月開花と星電導を選んだだけでは架空の反応ダメージが生成されないことをテストする。
- 全既存テスト、データ検証、実画面を確認する。
