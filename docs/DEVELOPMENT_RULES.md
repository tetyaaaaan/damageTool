# 開発規則

このファイルは、ダメージ計算ツールを継続して改良するための共通ルールです。

## 命名規則

- HTMLの `id` は既存互換を優先し、計算処理から参照されている名前は不用意に変更しません。
- 新しく追加する `class` は役割が分かる名前にします。例: `result-card`, `input-grid`, `formula-note`
- JavaScriptの変数名は意味が分かる英語にします。例: `baseDamage`, `damageBonusRate`, `enemyDefenseRate`
- CSV由来のDOM IDは、既存ルールに合わせて接頭辞を付けます。
  - キャラ: `cha_`
  - 光円錐: `wea_`
  - 遺物: `arti_`
  - 合計値: `sumStatus_`

## コメント規則

- 処理をそのまま日本語にしただけのコメントは増やしません。
- コメントは「なぜこの処理が必要か」「どのデータ仕様に依存しているか」を説明する場合に書きます。
- 未完成を示す文言は、公開ページや本番コードでは使いません。
  - 避ける例: `まだ作ってません`, `多分`, `精度は悪いです`
  - 使う例: `現在は通常ダメージのみ対応`, `計算対象外の補正があります`

## CSV規則

- 文字コードはUTF-8を使用します。
- 1行目は必ずヘッダー行にします。
- 不要な末尾カンマや空ヘッダーは残しません。
- 数値列は単位を入れず、数値だけを記録します。
  - 良い例: `12`
  - 避ける例: `12%`
- 説明文に半角カンマ `,` は使いません。現在の読込処理は簡易CSVパーサーのため、半角カンマが入ると列ずれします。
- 説明文の区切りには読点 `、` または句点 `。` を使います。
- CSVを更新した場合は、画面でキャラ・光円錐・遺物の選択結果が崩れていないか確認します。

## AdSense向け表記方針

- ページタイトルや説明文に「予定」「精度は悪い」など未完成に見える表現を使いません。
- 計算対象外の仕様がある場合は、免責や注意書きとして丁寧に記載します。
- 外部ゲーム名を扱う場合は、非公式ファンツールであることを明記します。
