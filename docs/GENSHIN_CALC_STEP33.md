# STEP33 武器効果の構造化

## 目的

武器効果を原文推測だけに依存せず、効果グループ、発動方法、適用対象、入力方針を明示して表示と計算を一致させる。

## 構成

- `weapon-effect-registry.json` は、確認済み武器の効果グループと補正IDの対応を保持する。
- 既存の `weapon-modifiers.json` は数値データとして維持し、計算時にレジストリの定義を重ねる。
- 未移行武器にも安定したフォールバックグループを付与し、全武器を同じカード構造で表示する。
- `inputPolicy` は `reflected`、`calculate`、`sourceContext`、`displayOnly` に分ける。
- `targetOwner` は `self`、`team`、`activeCharacter`、`enemy` に分け、装備者以外の効果を自己加算しない。

## 代表武器

- アモスの弓: 常時効果と飛翔時間を別グループ化する。
- 鐘の剣: シールド中のダメージ補正を明示トグルにする。
- 霧切の廻光: 入力反映済みの基礎補正と巴紋の層数を分離する。
- 流浪楽章: 三つの楽章を排他的な選択肢にする。
- 絶弦: 元素スキル・元素爆発へ常時自動適用する。
- 黒蝕: 元素爆発限定会心とチーム側効果を分離する。
- 実りの鉤鉈: 落下攻撃限定会心と命中後の段階を分離する。

## 重複処理

- 同一内容の完全重複レコードは後続レコードを隔離する。
- 汎用会心と攻撃種別限定会心が重なる場合は、限定レコードを正とする。
- 会心効果から誤生成されたダメージバフレコードは隔離する。
- 明示レジストリの `modifierOverrides` を最優先する。

## 監査

`node scripts/genshinWeaponEffectAudit.cjs` で全武器を分類し、次を `reports/genshin-weapon-effect-audit.*` に保存する。

- 全武器・全補正数
- 明示構造化済み件数
- フォールバック分類件数
- 入力方針別件数
- 同一説明・同一カテゴリの重複候補

## 継続方針

全武器はフォールバックを含めて分類済みだが、明示レジストリへの移行は代表武器から段階的に進める。移行済み武器では原文ヒューリスティックより `inputPolicy` と `activation` を優先する。
