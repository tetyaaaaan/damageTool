# 原神ダメージ計算 STEP40〜46 設計

## 目的

STEP40〜43で敵耐性・耐性デバフ・防御補正を構造化計算へ統合した。STEP44〜46では従来計算を廃止し、構造化計算を画面上の唯一の計算経路にする。

## STEP44 従来計算の廃止

- `atk`、`talent`、`ele_d` など従来計算専用入力を画面から削除する。
- `calc()` を呼ぶ従来計算ボタンと `main.js` を削除する。
- UID反映処理は新しいステータス入力だけを更新し、削除済み入力へ同期しない。
- 構造化エンジン内の旧形式コンテキスト変換は、既存データ・単体テスト用の境界アダプターとして残す。

## STEP45 構造化計算の本番昇格

- ユーザー向けの名称を「JSON計算」から「ダメージ計算」へ変更する。
- 計算ボタンは画面上に一つだけ置き、構造化エンジンを実行する。
- 警告と計算結果は右側の結果パネルに描画する。
- 内部IDは条件UI・E2E・外部連携の安定性を保つため、現段階では変更しない。

## STEP46 互換UIの清掃と移行ガード

- 従来計算用の静的結果、未実装表示、参考用聖遺物モーダルを削除する。
- 参照されなくなった `artifact.js` を削除する。
- 移行契約テストで、旧入力・旧ボタン・旧スクリプト・ユーザー向けJSON表記が存在しないことを固定する。
- STEP41〜43の耐性・防御計算テストと全Genshinテストを回帰基準とする。

## 完了条件

1. 計算実行ボタンが一つだけ存在する。
2. 実行時に条件を再構築してから構造化計算を行う。
3. 結果が右側の結果パネルへ表示される。
4. 従来計算のDOM IDとスクリプト参照が残っていない。
5. Genshin全テストとブラウザE2Eが成功する。
