# 原神 JSON 計算：補正条件カード

## 目的

計算結果を変える条件だけを、効果の出所と説明が分かる形でユーザーに提示する。
抽象的な「詳細条件」「装備枠補正」にはまとめず、各入力を実際の武器・聖遺物・天賦・命ノ星座の効果に所属させる。

## 表示順

補正条件は次の5カードを常に縦一列で表示する。

1. 元素反応
2. 武器補正
3. 聖遺物補正
4. 天賦補正
5. 命ノ星座補正

効果がないカードも省略せず、「該当する計算補正はありません」などの状態を表示する。
これにより、別のキャラクターや装備を試したときも、補正の有無を同じ位置で確認できる。

## 効果行に表示する情報

各効果行は次の情報を持つ。

- 効果名
- 発動条件と計算内容の説明
- 状態（自動適用、入力に反映済み、条件を選択、入力不足、表示のみ）
- ユーザーが決める必要がある入力またはチェック
- 現在の計算への反映量

入力欄は、それを要求する効果行の中にだけ表示する。記録治療量、補正提供者の能力値、リソース層数、対象数、スタック数なども独立した「専用参照入力」欄にはしない。

## 個別ルール

- アモスの弓は、常時効果と飛翔時間による追加効果を別の効果行にする。飛翔時間段階は武器カード内に置き、0段を「追加なし」と明記する。
- 聖遺物は、効果なし・入力ステータスへ反映済み・条件選択が必要、の状態を同じカードで区別する。
- 天賦は「キャラ・天賦補正」という一括チェックにせず、天賦名と説明を表示する。
- 命ノ星座は解放段階のプルダウンを置かない。現在のキャラクター入力欄の凸数を表示し、計算に関係する星座効果だけに説明と条件操作を付ける。
- 条件を再読み込みするときも、画面の計算入力欄と効果ごとの入力を正として状態を組み立てる。UIDから得た古い値で上書きしない。

## 検証

`node scripts/verifyGenshinCalc.cjs` で解析・条件カード・回帰テストを実行する。
`tests/genshin/genshinBrowserSmoke.e2e.cjs` では、甘雨 C1 とアモスの弓を例に、カード順、効果の所属、旧汎用入力の不在をブラウザー上で確認する。
