# STEP34〜36 元素反応・反応補正・残存天賦の設計

## 目的

現在の元素反応選択は、溶解・蒸発だけが攻撃本体へ適用され、それ以外は選択できても計算されない。STEP34〜36では、反応の計算方式、反応だけを対象にする補正、値なし天賦を同じ「special」へ押し込めず、構造化された別経路へ移す。

## 情報源と境界

- 通常反応のレベル固有値と反応係数は、バージョン差分を確認できるレベルスケーリング表を固定して保持する。
- キャラクター固有効果量は、リポジトリ内の `character-talents.json` と `talent-scalings.json` を正とし、既存の固定済み `genshin-db` 由来データと照合する。
- 月反応・星電導は通常反応へ代入しない。複数キャラクターの寄与、基礎ダメージ向上、固有会心など専用仕様が揃うまで `dedicatedFormulaRequired` とする。
- 未対応状態でも「反応なし」と同じに見せない。選択した反応名、未計算理由、結果へ反映されないことを画面へ明示する。

## STEP34 反応定義と計算経路

`reaction-definitions.json` に次を持たせる。

- `family`: `none` / `amplifying` / `additive` / `transformative` / `shield` / `statusOnly` / `dedicated`
- `coefficient`: 反応係数
- `damageElement`: 耐性計算に使う元素。拡散だけはユーザーが拡散元素を選ぶ。
- `emFormula`: 元素熟知補正式
- `critPolicy`: 通常は不可。明示された反応会心補正だけを許可する。
- `calculationStatus`: `supported` / `displayOnly` / `dedicatedFormulaRequired`
- `descriptionJa` / `unsupportedReasonJa`

計算経路は以下に分ける。

1. 増幅反応は攻撃本体へ倍率を掛ける。
2. 加算反応はレベル固有値を攻撃の基礎ダメージへ加える。
3. 変化反応は攻撃本体と別の反応結果を1件生成し、防御補正を使わず反応元素耐性だけを使う。
4. 結晶はダメージではなくシールド量を返す。
5. 凍結・原激化は状態のみで、数値結果を生成しない。
6. 月反応・星電導は専用式不足として停止する。

## STEP35 反応補正

`reactionBonus` は `applyTo` を反応IDへ正規化してから、現在選択中の反応だけへ適用する。汎用 `reactionDamageBonus` は全反応、`bloomDamageBonus` 等は該当反応だけ、`lunar*DamageBonus` は通常反応へ流用しない。

`reactionCritBonus` は通常会心率・会心ダメージへ混ぜず、反応結果だけへ適用する。ナヒーダC2は燃焼・開花・超開花・烈開花の固定会心と月開花会心を分割し、夢見月瑞希C6は拡散だけを対象にする。

反応補正監査は、全レコードに対して次を保証する。

- 対象反応を最低1件解決できる。
- 通常反応・月反応・星電導を誤って横断しない。
- 同一効果の分割レコードを二重適用しない。

## STEP36 残存天賦

値なし40件のうち、専用天賦倍率で表現済みの19件は引き続き重複抑止する。残り21件は次へ分ける。

- 独立ダメージ: 万葉、煙緋、イネファ。
- 参照値から求めるダメージ・反応基礎補正: 万葉、雲菫、白朮、月兆／星光系など。
- 天賦レベル依存値: 心海、エウルア、シロネン。元倍率が確定しない値を推測しない。
- ダメージではない状態・リソース: ロエン。

構造化できたものは元レコードへ値、参照式、上限、対象を追加する。現在のデータだけでは数値を確定できないものは、一般的な `値情報がありません` 警告ではなく、効果名と不足している専用倍率を示す `displayOnly` レコードへ変換する。

## UI

- 選択肢名へ「未対応」を混ぜない。反応カードの状態バッジで表す。
- 反応なし: 「反応なしを選択中です。」
- 対応済み: 方式、係数、元素熟知補正、反応補正、耐性を表示する。
- 状態のみ: ダメージを発生させないことを表示する。
- 専用式不足: 反応名を保ったまま、結果へ未反映である理由を表示する。
- 変化反応は攻撃タブへ混ぜず「元素反応」タブへ出す。
- 結晶は「シールド量」として表示する。

## テスト

- Lv90・元素熟知0の各通常反応ゴールデン値。
- 増幅、加算、変化、結晶の計算順序。
- 変化反応が敵防御を参照せず、元素耐性を参照すること。
- 反応補正114件と反応会心2件の対象解決監査。
- 未対応の月反応・星電導を0ダメージとして成功扱いしないこと。
- 値なし天賦監査で一般的な `MISSING_VALUE` を残さないこと。
