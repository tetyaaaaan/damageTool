# 原神計算画面 見た目修復計画

更新日: 2026-07-20  
対象: 前回の却下UIを撤去し、明示済みの見た目要件だけを反映する。

## 方針

1. 却下されたUIへ追加CSSを重ねない。追加ブロック、DOM、JSを先に除去する。
2. 計算精度修正はUI差分から分離して残す。
3. 未決事項を仕様として実装しない。入力／結果切替、期待値の扱い、補正条件モーダル化は今回対象外。
4. モバイルを縮小PCとして扱わず、ただし画面固有の情報や独自操作も増やさない。
5. レイアウト用カードは外側パネルの一階層だけにする。内側の区分は見出し・余白・区切り線で表し、入力欄やボタンなど操作部品の枠線はカード階層に数えない。

## 今回の修正範囲

### 1. ヘッダ

モバイルは `メニュー / tetinet / テーマ切替` の一行構成とする。

- メニューは左端。
- テーマ切替は右端。
- `原神計算`は表示しない。
- メニューは左寄せのドロワーとして表示する。
- PCのブランド、ナビゲーション、テーマ切替は変更しない。

### 2. 計算画面の骨格

- 前回追加した入力／結果ワークスペースタブを撤去する。
- 980px以下は入力、結果の順の一列表示へ戻す。
- 丸番号と丸い開閉ボタンを撤去する。
- キャラ・装備、ステータス、敵・反応設定は常時表示する。
- スマホでは外側パネルの二重の枠・影を弱め、セクションは罫線と余白で分ける。

### 3. UID読み込み

- `details`化を撤去し、通常の機能ブロックへ戻す。
- 重複見出し・長い補足をスマホで省略する。
- ラベル、入力、検索ボタンを一つのコンパクトなグリッドにする。
- 空のメッセージ領域へ高さを予約しない。
- 入力とボタンは他の計算入力と同じ16px、44px高を使う。

### 4. キャラ・武器・聖遺物選択

- タッチ端末では検索欄へ自動フォーカスしない。
- モーダルは画面端から12px以上離し、高さは最大 `calc(100dvh - 32px)` とする。
- 検索欄とクリアを同じ行に置く。
- 元素フィルタと解除を同じ横スクロール行に置く。
- スマホはキャラ名を一行、省略記号ありとし、サブ情報は元素・レアリティまでに抑える。
- 武器種は武器一覧に表示しない。
- 画像のlazy loading、async decoding、フォールバックを維持する。

### 5. 結果

- 前回追加した期待値表示切替を撤去し、元の結果項目へ戻す。
- 上位カテゴリは改行禁止とし、横スクロールを許可する。
- 次のタブが存在すると分かるよう、末尾に余白を置き、スクロール領域を画面端まで使う。
- 「通常・重撃・落下」を途中で改行しない。
- 攻撃名と数値の意味単位内では改行しない。攻撃名と数値領域は別段として扱う。
- 縦型カテゴリ等への変更は、別モック承認後に行う。

## 実装手順

1. `tetinet.css`の前回追加ブロックを削除する。
2. `genshinMobileWorkspace.js`と対応DOMを削除する。
3. 結果の期待値切替追加を削除する。
4. ヘッダDOMを左メニュー、ブランド、右テーマの順へ修正する。
5. 修復用CSSをファイル末尾の一か所へ追加する。
6. 既存テストを修復後の契約へ更新する。
7. 360px、390px、430px、PCで表示と操作を確認する。

## 受け入れ条件

- ヘッダが一行で、左端がメニュー、右端がテーマ切替になっている。
- ヘッダにページ固有文言がない。
- 丸番号、丸い開閉ボタン、入力／結果切替が残っていない。
- UIDに不自然な空白がない。
- 選択モーダルの周囲に背景が見える。
- 検索欄は16pxで、自動フォーカスされない。
- `通常・重撃・落下`が途中改行されない。
- 360pxで本文の横スクロールが発生しない。
- PCの二列レイアウトを維持する。
